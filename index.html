Echo Weaver is an immersive, first-person WebXR defense game built around a single design north star: your hands are the only controller. All gameplay emerges from natural hand gestures detected via the WebXR Hand Input API. Players stand in a mystical circular chamber and protect a central Heart Stone from waves of corrupted energy orbs — catching them bare-handed and weaving protective spells through mid-air sigil gestures.

The game is designed to feel like a magic lesson: intuitive enough to begin playing within 60 seconds, yet deep enough to reward practiced dexterity and strategic spell sequencing across extended sessions.

Core Fantasy

"You are an Echo Weaver — one of a forgotten order that does not cast spells at the world, but through it. Chaos already exists in the energy around you. Your gift is the ability to catch that chaos, shape it with intent, and release it back as order."

30-Second Core Loop

1 · Sense

Corrupted orbs spawn at the chamber perimeter and drift toward the Heart Stone.

2 · Catch

Raise open palm. An orb adheres to your hand on contact — one per hand simultaneously.

3 · Weave

Extend index finger, trace a sigil (○ — △) in the air to cast a spell.

4 · Defend

Spell activates, orb is consumed. Repeat before the Heart Stone falls.

Design Principles

Zero-Button Promise: The game never requires a button press, trigger pull, or joystick movement. If a mechanic requires a button, the mechanic changes instead.
Failure is Forgiving: Losing a held orb, missing a gesture, or taking Heart Stone damage should feel like a near-miss — not a punishment. Generous timers, clear visual warnings.
Show, Don't Tell: All player guidance is environmental. Glowing palm lines, ghost sigils, and particle beacons teach without text or tutorial overlays.

Architecture

Echo Weaver uses an Entity-Component pattern with a central GameManager singleton orchestrating the main loop. WebXR session lifecycle is decoupled from game logic to allow hot-reloading during development and a clean shutdown path on session end.

Frame Loop

// XRSessionManager.ts — registered as the XRSession animation-frame callback
function onXRFrame(time: number, frame: XRFrame) {
  session.requestAnimationFrame(onXRFrame);            // re-queue immediately (top priority)

  const dt = Math.min((time - lastTime) / 1000, 0.05); // cap at 50ms to handle tab-hidden spikes
  lastTime = time;

  // Execution order is strict — do not reorder
  HandTracker.update(frame, referenceSpace);            // 1. Raw joint data (highest priority)
  GestureRecognizer.update(dt);                        // 2. Sigil path recording + recognition
  GrabDetector.update(dt);                             // 3. Catch / release / fling logic
  GameManager.update(dt);                              // 4. Orbs, spells, wave progression
  CollisionManager.solve(dt);                          // 5. Physics resolution pass
  renderer.render(scene, camera);                      // 6. Draw to XR display
}

Hand Tracking

Hand tracking is the game's soul. The WebXR Hand Input API exposes 25 named joints per hand as XRJointSpace objects, each providing a 4×4 pose matrix (world-space position + orientation) and a radius hint. All 25 joints are sampled every animation frame inside onXRFrame before any game logic runs.

Palm Normal Derivation

The WebXR spec does not expose a dedicated palm joint. We derive the palm center and outward-facing normal each frame using three metacarpal positions and a cross product:

// HandTracker.ts
function getPalmData(hand: XRHand, frame: XRFrame, ref: XRReferenceSpace) {
  const pose  = (j: XRHandJoint) => frame.getJointPose(hand.get(j), ref)!;
  const toVec = (p: XRJointPose) =>
    new THREE.Vector3(p.transform.position.x, p.transform.position.y, p.transform.position.z);

  const wrist  = pose("wrist");
  const midMC  = pose("middle-finger-metacarpal");
  const idxMC  = pose("index-finger-metacarpal");
  const pnkMC  = pose("pinky-finger-metacarpal");

  // Palm center = midpoint of wrist and middle metacarpal base
  const palmCenter = toVec(wrist).lerp(toVec(midMC), 0.5);

  // Normal via cross product of two vectors lying in the palm plane
  const v1 = toVec(idxMC).sub(toVec(pnkMC));   // lateral axis (index → pinky)
  const v2 = toVec(wrist).sub(toVec(midMC));    // proximal axis (wrist → fingers)
  const normal = new THREE.Vector3().crossVectors(v1, v2).normalize();

  if (isRightHand) normal.negate();              // flip winding for right hand

  return { center: palmCenter, normal, radius: wrist.radius * 3.5 };
}

Open-Palm Pose Detection

// GrabDetector.ts — returns true when all four fingers are extended
function isOpenPalm(hand: HandData): boolean {
  return ["index", "middle", "ring", "pinky"].every(finger => {
    const tipDist  = hand.joints[`${finger}-finger-tip`].position
                       .distanceTo(hand.joints.wrist.position);
    const baseDist = hand.joints[`${finger}-finger-metacarpal`].position
                       .distanceTo(hand.joints.wrist.position);
    // Tip must be measurably farther from wrist than metacarpal base
    return tipDist > baseDist * 1.15; // 15% headroom absorbs tracking noise
  });
}

Tracking-Loss Graceful Degradation

Protocol — All timers reset on hand re-acquisition
0 – 100 ms Last-known pose held. No visual change. Player unlikely to notice.
100 – 500 ms Hand mesh fades to 30% opacity. Held orb detaches from hand and hovers in world-space at last-known position.
500 ms – 3 s Hand mesh hidden entirely. Orb continues hovering; emits a slow pulse-beacon of particles to mark its location.
3 s + Orb falls slowly (0.5 m/s²) and de-spawns on chamber floor contact. No damage dealt — timeout is intentionally generous.

Pointing Pose (Sigil-Trace Enable)

// HandTracker.ts — index extended, all others curled
function isPointing(hand: HandData): boolean {
  const indexExtended = (() => {
    const tip  = hand.joints["index-finger-tip"].position;
    const base = hand.joints["index-finger-metacarpal"].position;
    return tip.distanceTo(hand.joints.wrist.position) > base.distanceTo(hand.joints.wrist.position) * 1.2;
  })();

  const othersCurled = ["middle","ring","pinky"].every(f => {
    const tip  = hand.joints[`${f}-finger-tip`].position;
    const base = hand.joints[`${f}-finger-metacarpal`].position;
    return tip.distanceTo(hand.joints.wrist.position) < base.distanceTo(hand.joints.wrist.position) * 1.05;
  });

  return indexExtended && othersCurled;
}

Gesture System

The gesture pipeline uses a two-phase design: continuous Path Recording feeds a template-matching Recognizer on demand. The system must balance high recall (don't miss valid gestures) with high precision (no false positives during casual hand movement).

Phase 1 — Path Recording (PathSampler)

Recording activates only when the player state is orb-held + index-pointing. The index fingertip position is projected into camera-relative 2D NDC space — this normalises for body rotation and variable orb-holding distance from the headset.

// PathSampler.ts
const SAMPLE_HZ  = 20;       // one sample every 50 ms
const MAX_POINTS = 40;       // 2-second rolling window
const MIN_DELTA  = 0.008;    // skip near-stationary samples (NDC units)

class PathSampler {
  private pts: THREE.Vector2[] = [];
  private last = 0;

  update(tipWorld: THREE.Vector3, cam: THREE.Camera, t: number) {
    if (t - this.last < 1000 / SAMPLE_HZ) return;
    this.last = t;

    // Project 3D world position → 2D Normalised Device Coordinates (-1..+1)
    const ndc = tipWorld.clone().project(cam);
    const p   = new THREE.Vector2(ndc.x, ndc.y);

    // Only record meaningful movement
    if (!this.pts.length || p.distanceTo(this.pts.at(-1)!) > MIN_DELTA) {
      this.pts.push(p);
      if (this.pts.length > MAX_POINTS) this.pts.shift(); // evict oldest
    }
  }

  getPath() { return [...this.pts]; }
  reset()   { this.pts = []; }
}

Phase 2 — $1 Unistroke Recognizer

We implement a variant of the $1 Unistroke Recognizer (Wobbrock et al., 2007) — a well-studied algorithm proven robust for single-stroke recognition on low-sample-rate input. It normalises paths for scale, position, and rotation before template comparison via golden-section search.

// GestureRecognizer.ts
class DollarOneRecognizer {
  private readonly N    = 64;                      // resample to N equidistant points
  private readonly SZ   = 250;                     // scale into SZ×SZ bounding box
  private readonly ORI  = new THREE.Vector2(0, 0); // translate centroid to origin
  private readonly HALF = Math.sqrt(2) * this.SZ / 2; // half-diagonal of bounding box
  private readonly PHI  = 0.5 * (-1 + Math.sqrt(5));  // golden ratio for GSS

  recognize(raw: THREE.Vector2[]): { name: string; score: number } | null {
    if (raw.length < 10) return null;

    // Normalisation pipeline
    const resampled  = this.resample(raw, this.N);
    const radians    = this.indicativeAngle(resampled);
    const rotated    = this.rotateBy(resampled, -radians);
    const scaled     = this.scaleTo(rotated, this.SZ);
    const pts        = this.translateTo(scaled, this.ORI);

    // Template matching — golden-section search over ±180°
    let best = Infinity, bestName = "";
    for (const [name, tmpl] of Object.entries(TEMPLATES)) {
      const d = this.distanceAtBestAngle(pts, tmpl, -Math.PI, Math.PI, Math.PI / 90);
      if (d < best) { best = d; bestName = name; }
    }

    const score = 1 - best / this.HALF;
    return score > 0.72 ? { name: bestName, score } : null; // confidence threshold
  }

  // --- Helper stubs (full implementation in source) ---
  private resample(pts: THREE.Vector2[], n: number): THREE.Vector2[]        { return pts; }
  private indicativeAngle(pts: THREE.Vector2[]): number                     { return 0; }
  private rotateBy(pts: THREE.Vector2[], r: number): THREE.Vector2[]        { return pts; }
  private scaleTo(pts: THREE.Vector2[], sz: number): THREE.Vector2[]        { return pts; }
  private translateTo(pts: THREE.Vector2[], p: THREE.Vector2): THREE.Vector2[] { return pts; }
  private distanceAtBestAngle(a: THREE.Vector2[], b: THREE.Vector2[],
    lo: number, hi: number, thresh: number): number                          { return 0; }
}

Spellcasting

All spells inherit from a common Spell base class and are instantiated by SpellFactory upon gesture recognition success. The consuming orb is destroyed with a 300 ms dissolve animation before the spell activates — this intentional delay makes the cast feel weighted and deliberate.

// Spell.ts — base class for all Echo Weaver spells
abstract class Spell {
  protected lifeTime = 0;
  abstract readonly MAX_LIFE: number;

  constructor(protected scene: THREE.Scene, protected origin: THREE.Vector3) {
    this.origin = origin.clone();
    this.spawn();
  }

  abstract spawn(): void;             // set up geometry + audio
  abstract update(dt: number): void;  // advance simulation each frame
  abstract dispose(): void;           // clean up Three.js objects

  get isExpired(): boolean { return this.lifeTime >= this.MAX_LIFE; }
}

Implemented as a THREE.Raycaster against all active orb bounding spheres in a single frame. A visible CylinderGeometry beam mesh stretches from orb origin to hit point and tapers to 0 radius over 300 ms. Color: solar gold with an emissive bloom pass.

Duration1.2 s (expanding spherical wave)
OriginPlayer head position (camera world-space origin)
Wave radiusExpands 0 → 4 m over 1.2 s (eased cubic out)

Expanding SphereGeometry with additive blending and a pulsing opacity curve that peaks at 0.5 then fades to 0. Per-orb impulse each frame: force = (waveRadius − distToCenter) × 8.

Full Cast Sequence

// SpellFactory.ts — fires when GestureRecognizer emits a successful result
function onGestureRecognized(g: GestureResult, hand: HandData) {
  // 1. Haptic pulse (if XRHapticsActuator is available on this device)
  hand.source.gamepad?.hapticActuators[0]?.pulse(0.8, 120 /* ms */);

  // 2. Flash the sigil trail bright white for visual confirmation
  sigilTrail.flash(0xffffff, 200);

  // 3. Begin 300 ms orb dissolve animation
  hand.heldOrb?.beginConsumeAnimation(300);

  // 4. Spawn the spell after orb dissolve completes
  setTimeout(() => {
    const origin = hand.joints.wrist.position.clone();
    switch (g.name) {
      case "circle":   spells.push(new ShieldSpell(scene, origin));  break;
      case "line":     spells.push(new BeamSpell(scene, origin));    break;
      case "triangle": spells.push(new RepulseSpell(scene, origin)); break;
    }
    hand.heldOrb = null; // hand is now free to catch the next orb
  }, 300);
}

Emergency Gestures (No Orb Required)

Two-Handed Push — Emergency Deflect

Both palms face outward simultaneously; both hands make a rapid forward thrust (velocity > 0.8 m/s). Creates a weak gust: nearby orbs within 1.5 m are slowed 40% for 2 s. Single-use cooldown: 3 s.

Two-Handed Zoom — Situational Awareness

Both hands held up, palms facing each other. Pinch together or pull apart to scale the camera FOV (50°–90°). Useful for tracking distant threats. No cooldown.

Physics & Collision

Echo Weaver uses a bespoke, lightweight physics system with no external physics engine dependency. This keeps the bundle small and avoids WASM loading overhead on mobile-class XR hardware. All simulation is frame-rate independent via delta-time integration, and all forces are tuned for feel rather than physical accuracy.

Orb Movement Model

// Orb.ts — homing AI with configurable steering strength
class Orb {
  position  = new THREE.Vector3();
  velocity  = new THREE.Vector3();
  speed     = 0.25;                 // m/s base (increases each wave: +0.03 m/s per wave)
  readonly radius = 0.06;           // 6 cm sphere collider

  update(dt: number, heartPos: THREE.Vector3) {
    const toHeart = heartPos.clone().sub(this.position).normalize();
    const dist    = this.position.distanceTo(heartPos);

    // Homing coefficient: strong at distance, weakens in final approach
    // This prevents orbs swarming in a perfectly straight line (feels more alive)
    const k = THREE.MathUtils.mapLinear(dist, 0.5, 3.0, 0.40, 0.08);
    this.velocity.lerp(toHeart.multiplyScalar(this.speed), k * dt * 60);
    this.velocity.clampLength(0, this.speed * 1.2);    // cap velocity magnitude

    this.position.addScaledVector(this.velocity, dt);
    this.mesh.position.copy(this.position);
  }

  // --- Instability when held past the 5s safe window ---
  private holdTimer = 0;
  updateHeld(dt: number) {
    this.holdTimer += dt;
    const t = Math.min(this.holdTimer / 5.0, 1.0); // normalised 0 → 1

    if (t > 0.60) this.showInstabilityVFX(t);    // cracks appear, spark particles intensify
    if (t >= 1.0) this.forceEject();              // drop orb + area-of-effect warning pulse
  }
}

Collision Pairs — Narrowphase

Orb ↔ Palm

Sphere–sphere test each frame. CATCH event fires when:
① dist(orb, palmCenter) < orb.radius + palm.radius
② dot(palmNormal, toOrb) > 0.5 (palm is facing the orb)
③ isOpenPalm(hand) === true

Orb ↔ Heart Stone

Heart Stone collider radius = 0.15 m. On contact: deal −10 HP damage, destroy the orb with a burst particle effect, play the distorted crack SFX scaled by current HP.

Orb ↔ Shield Spell

Sphere–sphere against shield sphere. On contact: decrement shield HP, compute reflection vector (v − 2·dot(v,n)·n), apply to orb velocity with 0.6× restitution.

Orb ↔ Repulse Wave

Each frame: if dist(orb, waveOrigin) < currentWaveRadius, apply outward impulse: force = (waveRadius − dist) × 8 along normalised outward vector.

Spatial Hashing Broadphase

// CollisionManager.ts — cell size = 2× largest collider diameter
const CELL_SIZE = 0.15; // 15 cm grid cells

function worldToCell(p: THREE.Vector3): string {
  const cx = Math.floor(p.x / CELL_SIZE);
  const cy = Math.floor(p.y / CELL_SIZE);
  const cz = Math.floor(p.z / CELL_SIZE);
  return `${cx},${cy},${cz}`;
}

// Only pairs sharing a cell proceed to narrowphase sphere–sphere test.

Wave Announcement

Wave numbers are never shown as text. Instead, the Heart Stone emits a sequence of pulse-rings of light that travel outward along the chamber floor — one ring per wave number. Wave 3 → 3 sequential rings pulse from the Heart Stone base to the chamber wall, then fade. This communicates escalation through the world's visual language without any UI overlay.

Orb Instability Warning

When a held orb approaches its 5 s instability limit, the orb itself communicates the danger: surface cracks appear at t=60% of limit, particle sparks increase in density and speed at t=80%, and a low crackling audio cue begins at t=70%. At t=100% the orb ejects from the hand with a small area-of-effect shockwave that nudges the player (no HP damage, but disrupts any sigil trace in progress).

Audio Design

Audio is built on the Web Audio API with Howler.js managing asset loading, decoding, and playback. Spatial positioning uses PannerNode with the HRTF panning model for convincing 3D audio placement without a dedicated spatial audio library. Failure sounds are deliberately soft — never punishing or jarring.

Layered Soundtrack Architecture

Four Simultaneous Stems — Identical BPM · Shared Timeline
Layer 1 — DroneAlways active. Ambient pad texture. The silence baseline — always present but never intrusive.
Layer 2 — RhythmFades in at wave 2. Subtle, irregular percussion. Signals that threats are building.
Layer 3 — MelodyFades in when ≥4 orbs are simultaneously active. Urgent, ascending melodic line.
Layer 4 — TensionFades in when Heart Stone HP falls below 50%. Dissonant, frantic counter-melody.

All four stems are composed as 8-bar loops at a shared BPM and loaded pre-decoded into AudioBuffer objects. The shared AudioContext.currentTime timeline is established at session start; each layer is scheduled to its buffer offset to guarantee perfect alignment across all active stems regardless of when they're faded in.

Art Direction

Echo Weaver's aesthetic identity is Void Crystalline: geometric, luminous, and ancient. Low polygon counts are a deliberate design choice — shapes feel sacred and hand-carved from condensed light rather than manufactured. The dark environment exists purely as a stage that amplifies the luminous hands, orbs, and Heart Stone as the only actors.

Ghost Hand Shader

Fresnel Rim + Orb Proximity Tint
// GhostHand.frag (simplified GLSL)
uniform float orbProximity; // 0.0 = far away, 1.0 = touching palm
varying vec3  vNormal;
varying vec3  vViewDir;

void main() {
  // Fresnel: bright rim at silhouette edges, transparent at centre
  float fresnel  = pow(1.0 - clamp(dot(vNormal, vViewDir), 0.0, 1.0), 2.5);

  // Base colour shifts from cool blue → cyan as orb approaches
  vec3 baseColor = mix(vec3(0.20, 0.50, 1.0), vec3(0.0, 0.9, 1.0), orbProximity);

  // Overall alpha: semi-transparent, more visible at rim
  float alpha    = fresnel * 0.70 + 0.05;

  gl_FragColor   = vec4(baseColor * (1.0 + orbProximity * 0.5), alpha);
}

When holding an orb, the orb's colour additively tints the entire hand. Fingertips emit a persistent THREE.PointLight (intensity 0.4) that ramps to 1.0 during active sigil tracing — the hand literally glows as the player casts.

Orb Visual States

Free (Homing)

Jagged, semi-faceted geometry. Corrupt red emissive. Faint particle trail. Gentle rotation jitter.

Captured

Geometry smooths slightly. Transitions to verdant green. Gentle breathing pulse on scale.

Destabilising

Surface cracks appear via texture. Colour shifts gold → red. Spark particles increase. Scale jitter.

Definition of Done — Core Experience

A new user, given no verbal or textual instructions, can pick up an orb within 30 seconds, successfully cast at least one spell within 90 seconds, and — when asked afterward — describe the game's core mechanic as something that felt like "actual magic" rather than "gesture controls." The game must feel responsive, forgiving, and physically engaging. Every second of play should feel like a skill worth practising.
